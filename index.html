Trend Pemograman Website -dudu-


Pertama dari yang pertama



sebelum memulai semuanya kita harus mengerti 3 komponen dasar yaitu HTML, CSS, dan Javascript

Komponen web Itu hanya javascript! PWA? Juga JavaScript. FrameWork CSS? mereka hanya menggunakan CSS biasa.

Kita tidak perlu menguasai semua teknologi ini, setiap developer memiliki area pilihannya. Tapi kita perlu tahu bagaimana HTML, CSS dan JavaScript bekerja dan bagaimana mereka bekerja sama satu sama lain.


Node Js



Node.js adalah bahasa side-server, untuk memakainya kita perlu basic dari javaScript, dengan itu kita bisa menyelam lebih dalam ke bagian front-end dan back-end. Node Js adalah bahasa yang memberikan performa yang luar biasa. Fiturnya adalah memiliki ekosistem yang aktif dengan banyaknya paket pihak ketiga, dimana kita bisa memasukannya ke aplikasi kita. Yang membuatnya populer adalah karena semakin kesini semakin penting. Dan juga pada 2018 juga.



Front-End Frameworks



Framework pada FrontEnd contohnya seperti Angular, React atau Vue.js sangat membantu di bagian client javascript. Framework ini membantu kita untuk bisa membuat user experience yang menarik pada aplikasi web yang dikontrol menggunakan javascript. Ini sangat membantu dalam membuat user experience yang baik entah di mobile atau di pc.

Untuk mempelajarinya lebih dalam kita bisa melihat contohnya dari link-link dibawah :

  1. React
  2. Angular
  3. Vue.js


Static Page Generators




Bagaimana kita menyajikan pre-rendered version dari web kita tanpa harus pre-rendering secara dinamik ke suatu server? Jawabannya adalah static page generators. seperti Jekyll, Hugo atau project seperti di Nuxt.js untuk vue.js. Semua itu membantu kita mencapai 1 hal yaitu membuat halaman website yang bisa kita pre-render dalam server lokal kita atau dalam sebuah server.


Progressive Web Apps (PWAs)




berguna untuk menjumlahkan beberapa teknologi yang membantu kita dalam membangun sebuah web apps dan seperti native mobile apps.
Service Workers mengizinkan kita untuk mengontrol special browser cache. Untuk mendalaminya lebih dalam bisa melihat artikel ini PWA


Membangun Serverless Apps




maksud dari serverless adalah kita tidak perlu khawatir akan meng-manage dan meng-update server. Dimana skala kapasitasnya adalah tidak terbatas, dan kita hanya perlu membayar apa yang kita butuhkan.
contoh dari serverless apps adalah disini ServerLess FrameWork


Artificial Intelligence dan Bots




seakrang ini topic yang sering dibahas adalah AI dan Bot yang sangat membantu dan penting dalam kurun waktu kedepan ini. AI bukan hanya di mobil, tapi bisa dimana saja, bahkan bisa di website kita. Kita bisa mendalam kedalam Phyton, Machine Learning, Big Data, dan mendalami semua komponen AI.

Github Tutorial -dudu-


Pertama dari yang pertama



download dulu gitnya :

  1. Windows
  2. OSX
  3. Linux



Memulai Git




untuk memulai git kita perlu menginisiasi git tersebut di folder kita, dengan command seperti dibawah ini.

git init

nb : pastikan sudah berada di folder yang akan dikirim ke github di cmdnya


Add&Commit



Ketika kita menambahkan file, misalnya syalala.txt maka kita perlu menambahkannya di git, supaya masuk ke lognya maka perlu add&commit. perintahnya adalah git add(spasi)nama file dan commit -m "isinya" contohnya seperti dibawah ini bila kita menambahkan file syalala.txt:

add syalala.txt

lalu dilanjutkan dengan

commit -m "menambahkan file syalala.txt

maka itu artinya kita sudah meng-track file syalala.txt di git kita.


Menambahkan File ke Github




Untuk menambahkan file tersebut kita hanya perlu menggunakan:

git push origin master

nb: ini berlaku jika file yang mau kita kirim ada di branch master

karena kita akan menambahkan file kita ke server github, misalkan kita akan menambahkan file tersebut ke repository dengan link https://github.com/danielpartogi/hexo.git

maka untuk menambahkannya dengan perintah :

git remote add origin https://github.com/danielpartogi/hexo.git

baru dilanjutkan dengan perintah seperti diatas yaitu :

git push origin master



Branch




berguna untuk menyimpan file yang akan kita edit/perbaharui, dimana nanti file tersebut besifat tersembunyi. metode branch sangat berguna bila kita ingin kembali ke versi sebelumnya. Untuk menggunakannya dengan cara :

git checkout -b nama branchnya

contohnya

git checkout -b backed

bila ingin pindah branch hanya perlu menggunakan git checkout <nama filenya>


Pull & Merge




pull berguna untuk meng-update file local kita, dimana nanti file yang berada di github akan kita update terlebih dahulu ke directory kita. dengan command git pull Dan untuk menyatukan branch yang berbeda dengan branch yang saat ini kita menggunakan git merge <branch>


LOG




log melihat aktivitas yang telah dilakukan seseorang terhadap file yang ada. dapat dilihat dengan perintah

git log

maka nanti dia akan menampilkan seluruh aktivitas yang ada, serta kode commit dimana nanti kita bisa kembali ke versi log/aktivitas yang telah dibuat seseorang dengan perintah

git checkout <kodenya>

kodenya minimal hanya butuh 4 angka didepannya.


TAG




dengan tag kita bisa menandai versi dari file tersebut. contohnya kita ingin menambahkan 1.0.0 maka kita hanya perlu perintah

git tag 1.0.0 <kodelog>

kode log itu maksudnya adalah kode commit yang ada saat kita menggunakan git log.
